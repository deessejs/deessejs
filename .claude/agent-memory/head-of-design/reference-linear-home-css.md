# Linear home — CSS technique inventory

Verified 2026-06-26 from `https://linear.app` + `https://linear.app/pricing`.

## The "interactive mockup" hero (Linear's signature)

Linear's home hero is a **fully functional Linear UI rendered as a static mockup** — issue cards, status badges, the workspace sidebar, all visible. Not a screenshot — a live React component.

**What's in it (June 2026):**
- Top: "Coding Sessions" announcement badge
- Middle: A Linear issue card with status (`In Progress`, `High` priority), assignee, labels (`Performance`, `iOS`), cycle number, project name
- Right: A Slack/Linear activity feed showing issue triage, label addition, PR draft
- Floating: An "AI agent" panel showing "Examining the startup path... Thinking..." with a typing cursor
- Bottom: A "Linear created the issue via Slack" attribution

**The principle:** instead of telling users "Linear is for AI-era teams," **show them** by rendering a Linear issue being worked on by an AI agent. The hero IS the product demo.

**For DeesseJS:** the equivalent is rendering a **DeesseJS template dashboard** as the hero — an auth/billing/jobs UI generated by the template, with an agent cursor showing the agent "configuring" it.

## The "FIG" section label pattern

Linear uses a small, monospace label above each section to index the page:

```
FIG 0.1  Built for purpose
FIG 0.2  Powered by AI agents
FIG 0.3  Designed for speed
```

CSS:
```tsx
<div className="flex items-center gap-3">
  <span className="font-mono text-xs uppercase tracking-widest text-muted-foreground">
    FIG 0.{n}
  </span>
  <span className="h-px flex-1 bg-border/40" />
</div>
```

**Pattern:** the label is monospace, uppercase, tracked-widest, muted-foreground. A hairline rule extends from the label to the right edge of the container. This signals "documented, not decorative."

**For DeesseJS:** this is perfect for `/about` (Principles, Convictions, Roadmap) and `/pricing` (sections become FIG 0.1, FIG 0.2). The document/research vibe fits the brand.

## Hairline-bordered feature cards (Linear style)

Linear's pricing page features table uses hairline borders as the primary separator:

```css
.feature-row {
  display: grid;
  grid-template-columns: 1fr repeat(4, max-content);
  gap: 0;
  border-bottom: 1px solid oklch(0.5 0 0 / 0.1);
}

.feature-row > * {
  padding: 16px 24px;
}

.feature-row:first-child {
  font-weight: 500;
  text-transform: uppercase;
  font-size: 12px;
  letter-spacing: 0.04em;
  color: var(--muted-foreground);
}
```

**Rules:**
- NO row background fills (just dividers).
- NO padding on the row itself — padding is per-cell.
- The first row is a header (uppercase, mono, muted).
- Feature columns: label column wide-left, value columns narrow-right.

**For DeesseJS:** this is what `comparison-block.tsx` should look like (we're already close — just confirm the `border-b` rhythm and the per-cell padding).

## Color usage

Linear uses a slightly warmer palette than Vercel:
- Background: `oklch(0.99 0.005 80)` (very subtle warm white)
- Text: `oklch(0.18 0.005 60)` (warm black)
- Border: `oklch(0.18 0.005 60 / 0.08)`
- Accent: `oklch(0.55 0.18 250)` (focused blue, same as Vercel)

The warmth comes from a slight yellow tint in the neutrals — barely perceptible, but it makes the page feel less sterile.

**For DeesseJS:** consider tinting the neutrals slightly warm. Adjust `oklch(0.18 0 0)` → `oklch(0.18 0.003 60)` in the design tokens. The effect is subliminal but it works.

## Section composition

- Container: `max-w-[1280px]`
- Section padding: `py-24`
- Section transitions: hairline `border-b border-border/40` between sections
- The pricing page uses a sticky feature table (first column + first row both sticky)

## Motion

Linear uses motion sparingly:
- Hover: 120ms transitions on color/border only
- Status changes: 200ms with `cubic-bezier(0.4, 0, 0.2, 1)`
- No scroll-driven animations
- No autoplay anything

**Pattern:** motion is a state signal, not a flourish.

## The "agent activity" feed (Linear's AI positioning)

Linear shows an in-hero activity feed where:
- A human (`@karri`) types a comment
- The Linear agent (`@Linear`) responds with an analysis
- A PR is auto-created
- A status badge transitions from `Todo` to `In Progress`

This is **the same pattern as the Supabase Realtime hero** — a live mockup of the product doing the thing. The CSS is identical: a static React component with multiple children rendered to look dynamic.

**For DeesseJS:** the about page "Get in touch" section could end with a tiny activity feed showing "3 agents deployed this week" — pulled from real data once we have it, or hardcoded for v1.

## What Linear does NOT do

- No SVG illustrations on the home (the UI IS the illustration)
- No emoji in the UI (text status badges only)
- No gradients of any kind (their brand is pure neutrals)
- No drop shadows (depth via border contrast only)
