# Resend home — CSS technique inventory

Verified 2026-06-26 from `https://resend.com` (text fetch via fresh — Resend ships minimal HTML, the design is mostly in the rendering).

## The "email flow" hero (Resend's signature pattern)

The classic Resend home hero composition:
```
[ Terminal/Code Block ]  →  [ Arrow ]  →  [ Inbox Card ]
```
Three elements horizontally arranged with a hairline arrow between them. The terminal shows `import { Resend } from 'resend'; const resend = new Resend(...)`. The inbox card shows the resulting email rendered as HTML.

**Why this works for DeesseJS:** replace the three elements with:
- **Code block** (the template's package.json snippet)
- **Arrow** (a hairline arrow with `<Zap />` or `<Bot />` icon)
- **Output card** (the auth/billing UI generated by the template)

This is exactly the kind of "show what you ship" visual you asked for, without committing to a full animated dashboard mockup.

## CSS recipe

```tsx
<div className="grid grid-cols-[1fr_auto_1fr] items-center gap-6">
  {/* Code block */}
  <div className="rounded-lg border bg-card p-4 font-mono text-sm">
    <div className="text-muted-foreground">// your agent</div>
    <div>await <span className="text-foreground">deesse</span>.auth.<span className="text-foreground">signIn</span>(...)</div>
  </div>

  {/* Arrow with icon */}
  <div className="text-muted-foreground">
    <ArrowRight className="size-6" />
  </div>

  {/* Output card */}
  <div className="rounded-lg border bg-card p-4">
    <div className="text-xs text-muted-foreground">Response</div>
    <div className="font-mono text-sm">{`{ user: '...' }`}</div>
  </div>
</div>
```

**Composition rules (from Resend):**
- The arrow is the "story" — it must be visually heavier than the elements (24px icon, not 16px).
- The code block is a `<pre>` with `font-mono`, slightly muted background (`bg-muted/50`).
- The output card mirrors the code block's border + radius.
- All three elements share the SAME vertical center (`items-center`).

## Resend's color palette

Resend uses a tight, sharp palette:
- Background: pure white (`#FFFFFF`) / dark (`#0A0A0A`)
- Accent: black-on-white for everything interactive (no secondary color)
- Code highlight: monospace font weight 500, no syntax colors (mono palette only)

This is **closer to DeesseJS's brand** than Supabase's. The recipe above transfers 1:1.

## The "agent-first" framing

Resend's hero copy is structured around *what the agent does*:
- "Email for developers" → positioning
- "Email for agents" → callout (their /agents landing page exists)
- A list of "tools" an agent can use (CLI, MCP, Skills)

**For DeesseJS:** the home hero should mirror this — a primary headline + an "agent-callable" subhead + a list of tools the template ships. We've already done this in copy; the CSS is the missing piece.

## Interactive features (Resend hero patterns)

Resend does NOT do:
- Mesh gradients (rejected as too "marketing")
- Cursor-tracked glows
- Animated dashboards

Resend DOES do:
- Sharp 1px borders, no shadows
- Hairline arrows as composition elements
- Mono font for all code/data
- Inline icons (Lucide-style)

## Section composition

- Container: `max-w-[1200px]`
- Section padding: `py-20` (tighter than Vercel)
- Cards: 8px radius, no shadow, `border-border/40`
- Section transitions: subtle `border-b border-border/40` between sections

## The "tools" section pattern

Resend's `/agents` page lists tools in a vertical stack:
```
🔧 Command line tool     [Install →]
🔧 MCP Server            [Add →]
🔧 Skills                [Browse →]
```

Each row is a `<div>` with `flex items-center justify-between border-b border-border/40 py-4`. The icon is left, the title + description center, the CTA right.

**For DeesseJS:** mirror this for the "agents can call" section on `/about` — a vertical list of callable surfaces (auth, billing, jobs, storage, notifications, API) with the same row anatomy.
