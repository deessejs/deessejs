`apps/docs/` is the Fumadocs-powered documentation site. Lives in the same monorepo as `apps/web` but has its own dependencies and tooling.

## Stack

- `fumadocs-core` 16.10.5 — the rendering engine
- `fumadocs-mdx` 15.0.12 — MDX loader + collection generator
- `fumadocs-ui` 16.10.5 — default components
- Next.js 16.2.9, React 19.2.7, Tailwind 4.3.1, TypeScript 6.0.3
- No shared `@workspace/ui` consumption (uses its own `src/lib/cn.ts`)

## Scripts

- `pnpm --filter docs dev` — local dev with HMR
- `pnpm --filter docs build` — production build
- `pnpm --filter docs types:check` — `fumadocs-mdx && next typegen && tsc --noEmit` (postinstall runs the same)
- `pnpm --filter docs lint` — eslint

## Generated files

`apps/docs/.source/` is generated by `fumadocs-mdx` on `postinstall` and on dev/build. NEVER edit files in there — changes are wiped.

The loader entrypoints (from `src/lib/source.ts`) re-export from `.source/`. If you need to change which collections are exposed, edit `source.config.ts` (or the loader in `src/lib/source.ts`), not `.source/*`.

## Routes that are easy to miss

- `app/api/search/route.ts` — built-in search backend, no extra config
- `app/llms.txt/route.ts` + `llms-full.txt/route.ts` — for LLM discovery (recent Fumadocs convention, keep them)
- `app/og/docs/[...slug]/route.tsx` — generates per-page OG images on the fly (used by Twitter/LinkedIn unfurls)

## Cross-app links

- `apps/web/src/app/(unprotected)/(marketing)/page.tsx` links to `/docs` (the Fumadocs home). Make sure route changes in `apps/docs/app/(home)/page.tsx` don't break the source of the link target.
- `apps/web/src/components/homepage/code-preview.tsx` likely renders an IDE-style preview pointing at code that runs against the docs — coordinate visual changes.

**How to apply:** When touching `apps/docs/`, check the generated `.source/` first to see what's actually wired. Edit `source.config.ts` and `src/lib/source.ts`, not the generated files. Re-run `pnpm --filter docs types:check` after content schema changes.
