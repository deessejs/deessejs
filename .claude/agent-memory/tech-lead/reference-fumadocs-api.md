Reference for the Fumadocs docs framework as wired into `apps/docs/` (researched 2026-06-24). Use when scaffolding, customizing, or debugging apps/docs.

**Versions confirmed in apps/docs scaffold**: `fumadocs-core@16.10.5`, `fumadocs-ui@16.10.5`, `fumadocs-mdx@15.0.12`. Next.js 16.2.9, React 19.2.7, Tailwind v4.3.1.

## 1. Core architecture — 4 layers

### 1.1 Content layer (the "what")
- **`content/docs/*.mdx`** — page sources (frontmatter + MDX body)
- **`source.config.ts`** — declares collections + Zod schemas for frontmatter
  - `defineDocs({ dir, docs: { schema: pageSchema }, meta: { schema: metaSchema } })` — combo helper
  - `defineCollections({ type: 'doc'|'meta', dir, schema })` — primitive
  - `defineConfig({ mdxOptions })` — global MDX config (rehype/remark plugins)
- **`.source/`** — auto-generated by `fumadocs-mdx` (postinstall script in apps/docs/package.json). Regenerated on every `fumadocs-mdx` run. Do NOT edit.
- **`src/lib/source.ts`** — single entry point, exports `source = loader({...})`

### 1.2 Route layer (Next.js App Router)
| File | Purpose |
|---|---|
| `src/app/layout.tsx` | Root layout — mounts `<RootProvider>` + imports `global.css` |
| `src/app/(home)/layout.tsx` | Landing (no sidebar) — `<HomeLayout>` |
| `src/app/(home)/page.tsx` | Landing page |
| `src/app/docs/layout.tsx` | Docs shell — `<DocsLayout tree={source.getPageTree()}>` |
| `src/app/docs/[[...slug]]/page.tsx` | Catch-all dynamic doc resolver |
| `src/app/api/search/route.ts` | Search endpoint (Orama) — `createFromSource(source)` |
| `src/app/llms.txt/route.ts` | LLM-friendly sitemap |
| `src/app/llms.mdx/docs/[[...slug]]/route.ts` | Per-page markdown export for LLMs |
| `src/app/og/docs/[...slug]/route.tsx` | Auto OG images |
| `proxy.ts` | **Next.js 16** — replaces `middleware.ts`. Handles `.md` suffix → content negotiation |

### 1.3 Layout components (Fumadocs UI)
- `HomeLayout` from `fumadocs-ui/layouts/home` — landing
- `DocsLayout` from `fumadocs-ui/layouts/docs` — sidebar + TOC
- `DocsPage`, `DocsBody`, `DocsTitle`, `DocsDescription` from `fumadocs-ui/layouts/docs/page`
- `MarkdownCopyButton`, `ViewOptionsPopover` — UI controls inside doc pages
- `baseOptions()` in `src/lib/layout.shared.tsx` — shared nav/github config

#### 1.3.1 `DocsLayout` props (full reference)
```tsx
<DocsLayout
  // Core
  tree={source.getPageTree()}        // from loader() OR hardcoded object
  nav={{ title: '...', url: '...' }} // top nav config
  links={[...]}                       // top nav links
  githubUrl="..."                     // top-right GitHub icon link
  themeSwitch={{ enabled, ... }}      // light/dark toggle config
  searchToggle={{ enabled, ... }}     // search button config
  slots={{ ... }}                     // custom slots for nav

  // Layout Tabs (multi-product)
  tabs={false}                        // disable
  // OR explicit array:
  tabs={[
    { title: 'Template', url: '/docs/template', description: '...' },
    { title: 'Cloud',    url: '/docs/cloud',    urls: new Set([...]) },
  ]}
  // OR rely on meta.json { root: true } in folders
  tabs={{ transform: (option, node) => ({ ...option, icon: <MyIcon /> }) }}

  // Sidebar options
  sidebar={{
    enabled: true,                    // default true
    collapsible: true,                // collapse on desktop
    defaultOpenLevel: 0,              // folders auto-open up to level N (1-based)
    prefetch: true,                   // ⚠️ set false for Vercel (cost warning)
    banner: <CustomBanner />,         // top of sidebar
    components: {                     // override individual sidebar parts
      Separator: SidebarSeparator,
    },
  }}
>
```
- `i18n` prop is **deprecated** — use `<RootProvider i18n={i18nProvider(...)}>` instead.
- `nav.title` accepts JSX (e.g. logo + text).
- `links` are external/internal nav items shown in the header.

#### 1.3.2 The Layout System (philosophy)
From Fumadocs docs: layout is built around 5 principles — composable, flexible, cohesive, predictable, compatible (Baseline Widely Available CSS). Implication: avoid hardcoded heights, components respond to each other (sidebar collapse animates), works on older browsers.

### 1.4 Styling
`src/app/global.css`:
```css
@import 'tailwindcss';
@import 'fumadocs-ui/css/<theme>.css';   /* neutral | shadcn | … */
@import 'fumadocs-ui/css/preset.css';    /* animations + utilities */
```
- Only Tailwind v4 supported
- `shadcn` preset adopts shadcn UI colors automatically (compatible with [[project-shadcn-final-state]] packages/ui)

### 1.5 DeesseJS content structure (planned 2026-06-24)
Per [[project-apps-docs-stack-decision#2. Layout Tabs — Template (paid) + Lite (free) from v0]], apps/docs will host two product docs under one site using Layout Tabs:
```
apps/docs/content/docs/
├── index.mdx                    # landing / "What is DeesseJS?"
├── (shared)/                    # route group — pages available in both tabs
│   ├── why-deessejs.mdx
│   └── getting-started.mdx
├── template/                    # Template tab content (paid product)
│   ├── meta.json                # folder metadata + tab config (optional)
│   ├── index.mdx                # Template overview
│   ├── architecture.mdx
│   ├── packages/
│   │   ├── ui.mdx
│   │   ├── auth.mdx
│   │   └── mail.mdx
│   └── api.mdx
└── lite/                        # Lite tab content (free starter)
    ├── meta.json
    ├── index.mdx
    ├── architecture.mdx
    └── getting-started.mdx
```
URLs:
- `/docs/template/...` → Template tab
- `/docs/lite/...` → Lite tab
- `/docs/(shared)/...` → accessible from both (slugs don't include `(shared)` because of route group)

## 2. Key APIs (3 things to remember)

### 2.1 RootProvider — wrap the whole app
```tsx
import { RootProvider } from 'fumadocs-ui/provider/next';

<RootProvider
  theme={{ enabled: false }}   // disable next-themes (DESIGN.md §1.6 — see gotchas)
  search={{ enabled: false }}  // disable Orama if replacing with custom
  i18n={i18nProvider(translations, lang)}   // when i18n active
>{children}</RootProvider>
```
Body MUST have `className="flex flex-col min-h-screen"` (required for layout).

### 2.2 Loader API — query content (src/lib/source.ts)
```ts
import { loader } from 'fumadocs-core/source';
import { docs } from 'collections/server';

export const source = loader({
  baseUrl: '/docs',           // or 'docsRoute' from shared.ts
  source: docs.toFumadocsSource(),
  plugins: [lucideIconsPlugin()],   // resolves lucide icon names in frontmatter
  i18n,                        // when i18n active
});

source.getPage(['slug'])       // single page (with body, toc, frontmatter)
source.getPages()              // all pages
source.getPageTree()           // nav tree (sidebar data)
source.getPageTree('fr')       // per-locale tree (i18n)
source.generateParams()        // for Next.js generateStaticParams
page.data.body                 // compiled MDX (React Server Component)
page.data.toc                  // TOC structure
page.data.frontmatter          // Zod-validated frontmatter
```
Note: `lucideIconsPlugin` resolves string icon names to lucide-react components — confirms `lucide-react` dep is intentional.

### 2.3 Search API — auto-fed from source
```ts
// src/app/api/search/route.ts
import { createFromSource } from 'fumadocs-core/search/server';
export const { GET } = createFromSource(source, { language: 'english' });
```
Search client uses `useDocsSearch()` from `fumadocs-core/search/client`. Default search engine is Orama (WASM in-browser, no backend).

`language` options: `'english'`, `'french'`, `'german'`, `'spanish'`, etc. (Orama supported languages) — `'multilingual'` also possible.

## 3. MDX components — full inventory

### 3.1 Built-in (always available via `fumadocs-ui/mdx`)
- **`<Callout>`** — types : `info` (default) / `warn`/`warning` / `error` / `success` / `idea`. Accepts `title` prop.
- **`<Cards>` + `<Card>`** — visual link grid. Props: `title`, `href`, icon, children. `href` optional.
- **Code Blocks** — via `rehypeCode` Shiki
- **Headings** (h1-h4) — auto-anchor (`Hello World` → `#hello-world`)
- **`<a>`** — auto-detects internal vs external. Internal uses framework `Link` (prefetch). External gets `rel="noreferrer noopener" target="_blank"`.

### 3.2 Additional components (must import in `getMDXComponents`)
- **`<Tabs>` + `<Tab>`** — Radix-based. Features:
  - `items={['A', 'B']}` array
  - `groupId="lang"` — shared value across all tabs with same id (sessionStorage)
  - `persist` — promote groupId storage to localStorage
  - `defaultIndex={n}` — initial active tab
  - `id="tab-xxx"` + URL hash — link directly to a tab
  - `updateAnchor` — auto-update URL hash on tab change
- **`<Accordions>` + `<Accordion>`** — Radix-based. `type="single"`, `title`, `id` (linkable). Default value = title unless `id` provided.
- **`<Steps>`** — numbered sections
- **`<Files>`** — display file structure
- **`<Banner>`** — top-of-site banner
- **`<Zoomable Image>`** — zoom-in on images
- **`<Inline TOC>`** — TOC inline in content (vs auto right-side)
- **`<Type Table>` + `<Auto Type Table>`** — type documentation tables
- **`<GitHub Info>`** — repo info card
- **`<Graph View>`** — all-pages graph
- **Code Block (Dynamic)** — runtime syntax highlighting (vs build-time Shiki)

Import pattern:
```tsx
import * as TabsComponents from 'fumadocs-ui/components/tabs';
import * as AccordionComponents from 'fumadocs-ui/components/accordion';
import { CodeBlock, Pre } from 'fumadocs-ui/components/codeblock';

export function getMDXComponents(components?: MDXComponents) {
  return { ...defaultMdxComponents, ...TabsComponents, ...AccordionComponents, ...components };
}
```

### 3.3 Link utilities
- **`createRelativeLink(source, page)`** — override `<a>` to enable `[link](./file.mdx)` in MDX
- **`getPageTreePeers(tree, url)`** — render "related pages" cards at end of article

### 3.4 Override `pre` for Fumadocs CodeBlock UI
```tsx
import { CodeBlock, Pre } from 'fumadocs-ui/components/codeblock';
pre: ({ ref: _ref, ...props }) => (
  <CodeBlock {...props}><Pre>{props.children}</Pre></CodeBlock>
)
```
`CodeBlock` props: `keepBackground` (preserve Shiki bg), `icon` (custom lang icon).

## 4. Navigation — page conventions & meta.json

### 4.1 File → page slugs (auto)
| Path (relative to `content/`) | Slugs |
|---|---|
| `./dir/page.mdx` | `['dir', 'page']` |
| `./dir/index.mdx` | `['dir']` (folder's index) |
| `./(group-name)/page.mdx` | `['page']` — parens mean "don't change child slugs" |

Frontmatter detected by Fumadocs:
- `title` — page title (h1)
- `description` — meta + page description
- `icon` — string resolved by `loader`'s `icon` handler (e.g. lucideIconsPlugin)

### 4.2 Folder → meta.json
```json
{
  "title": "Display Name",
  "icon": "MyIcon",
  "defaultOpen": true,
  "collapsible": true,
  "pages": ["index", "getting-started"],
  "pagesIndex": "overview",
  "root": true       // ← root folder = Layout Tab (only one visible at a time)
}
```

### 4.3 `pages` array — rich syntax
- `"index"` — file reference
- `"---Label---"` — separator. Icon: `"---[Icon]Label---"`
- `"[Text](url)"` — link. Icon: `"[Icon][Text](url)"`
- `"external:[Text](url)"` — explicit external link
- `"..."` — rest of items, alphabetical
- `"z...a"` — rest reversed
- `"...folder"` — extract from a subfolder
- `"!item"` — exclude from `...` or `z...a`
- **Rule**: no duplicate URL across the entire page tree (Fumadocs uses URL for active state)

### 4.4 Root folders = Layout Tabs (3 ways to enable)

**Way 1** — Folder `meta.json`:
```json
{ "title": "Template", "root": true }
```

**Way 2** — Explicit array in DocsLayout (cleaner, no content coupling):
```tsx
<DocsLayout tabs={[
  { title: 'Template', url: '/docs/template', description: '...' },
  { title: 'Cloud',    url: '/docs/cloud' },
  { title: 'API',      url: '/docs/api' },
]} />
```
- `url` activates the tab for that path + sub-paths
- `urls: new Set([...])` for multi-URL activation
- `transform` callback to customize each tab (icon, etc.)

**Way 3** — Disable entirely:
```tsx
<DocsLayout tabs={false} />
```

By default the tab trigger renders as a **Dropdown** (hidden unless one tab's item is active). Useful behavior — single-product sites don't need it visible.

When `"root": true`:
- Fumadocs UI shows folder as a top-level tab
- Only the opened tab's children appear in sidebar
- **Use case**: separate products in one doc site (e.g. Template / Cloud / API)
- DeesseJS doesn't need this yet (1 product) but useful for v2+
- Strategy note: explicit `tabs` array (Way 2) is preferred for DeesseJS because it doesn't couple content structure to navigation decisions — content can be reorganized without breaking nav.

## 5. i18n (multilingual)

### 5.1 Setup (Next.js)
```ts
// src/lib/i18n.ts
import { defineI18n } from 'fumadocs-core/i18n';
export const i18n = defineI18n({
  defaultLanguage: 'en',
  languages: ['en', 'fr'],
  hidePrefix: 'always',     // default locale has no /en prefix
});
```

### 5.2 Middleware
```ts
// proxy.ts (Next.js 16) or middleware.ts
import { createI18nMiddleware } from 'fumadocs-core/i18n/middleware';
import { i18n } from '@/lib/i18n';
export default createI18nMiddleware(i18n);

export const config = {
  matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
};
```

### 5.3 Routing
- Move `app/(home)`, `app/docs/`, etc. → `app/[lang]/...`
- `loader({ i18n, ... })` for source
- `source.getPageTree('fr')`, `source.getPage(slug, 'fr')`
- RootProvider: `<RootProvider i18n={i18nProvider(translations, lang)}>`

### 5.4 Translations (UI strings)
```ts
import { uiTranslations } from 'fumadocs-ui/i18n';
export const translations = i18n.translations()
  .extend(uiTranslations())             // Fumadocs UI strings (Search, On this page, etc.)
  .add({
    en: { displayName: 'English' },
    fr: { displayName: 'Français' },
  });
```

### 5.5 baseOptions can vary per locale
```ts
export function baseOptions(locale: string): BaseLayoutProps {
  return { nav: { title: locale === 'fr' ? 'DeesseJS' : 'DeesseJS' }, /* … */ };
}
```

### 5.6 Notes
- Fumadocs handles **routes + nav only**, NOT page content translation
- Compatible with next-intl, react-i18next, etc. for actual content
- `fallbackLanguage` — pages missing in a locale fall back to default

## 6. fumadocs-openapi (API reference generation)

### 6.1 Architecture
```ts
// lib/openapi.ts (server)
import { createOpenAPI } from 'fumadocs-openapi/server';
export const openapi = createOpenAPI({
  input: ['./openapi.json'],   // or external URL
});
```

```tsx
// components/api-page.tsx (client)
'use client';
import { createOpenAPIPage } from 'fumadocs-openapi/ui';
export const OpenAPIPage = createOpenAPIPage();
```

```ts
// lib/source.ts — adds badge in page tree
import { openapiPlugin } from 'fumadocs-openapi/server';
export const source = loader({ plugins: [openapiPlugin()], ... });
```

### 6.2 Mode A — generate static MDX files (recommended)
```js
// scripts/generate-docs.mjs
import { generateFiles } from 'fumadocs-openapi';
import { openapi } from '@/lib/openapi';
void generateFiles({
  input: openapi,
  output: './content/docs/api',
  includeDescription: true,   // careful: endpoint descriptions must be valid MDX
});
```
→ Run via `pnpm generate-docs`. Produces `.mdx` per endpoint, organized by tag.

### 6.3 Mode B — Loader API direct (dynamic, no files)
```ts
import { loader } from 'fumadocs-core/source';
import { openapi } from '@/lib/openapi';
export const source = loader({ plugins: [openapiPlugin()], /* ... */ });
```
→ Resolves OpenAPI at request time. More flexible, more complex.

### 6.4 Wire OpenAPIPage into MDX
```tsx
// in dynamic page renderer
<MDX components={getMDXComponents({
  OpenAPIPage: async (props) => (
    <OpenAPIPage {...await openapi.preloadOpenAPIPage(page)} {...props} />
  ),
})} />
```

### 6.5 CSS (mandatory)
```css
@import 'tailwindcss';
@import 'fumadocs-ui/css/<theme>.css';
@import 'fumadocs-ui/css/preset.css';
@import 'fumadocs-openapi/css/preset.css';   /* ← adds API UI styles */
```

### 6.6 Bridge with oRPC
oRPC has `OpenAPIHandler` that emits OpenAPI 3.x spec. Pass its output to `createOpenAPI({ input })` — direct compatibility, no glue code.

### 6.7 DeesseJS context
[[apps/template/apps/api]] uses oRPC. When it has a stable spec, generating API reference becomes 1 script + 1 import. Not in v0.0.1 (per [[project-apps-docs-stack-decision]]).

## 7. Code blocks & syntax highlighting

### 7.1 Engine: Shiki via `rehypeCode`
```ts
// source.config.ts
import { defineConfig } from 'fumadocs-mdx/config';
export default defineConfig({
  mdxOptions: {
    rehypeCodeOptions: {
      themes: { light: 'github-light', dark: 'github-dark' },
      // inline: 'tailing-curly-colon',   // opt-in inline `code{lang}`
      transformers: [...rehypeCodeDefaultOptions.transformers],
    },
  },
});
```

### 7.2 Meta string in code fences
- ` ```ts title="Title" ` — adds title attribute to `<pre>`
- ` ```ts icon=false ` — disable per-block icon
- `filterMetaString` option to customize parsing

### 7.3 Override `pre` for Fumadocs UI
Already shown in §3.4. The `CodeBlock` component provides:
- Copy button
- Title display
- Language icon
- `keepBackground` — keep Shiki's color tokens (vs Fumadocs override)

### 7.4 Twoslash (TypeScript hovers in code)
```bash
pnpm add fumadocs-twoslash twoslash
```

**Next.js gotcha** (mandatory):
```js
// next.config.mjs
const config = {
  serverExternalPackages: ['typescript', 'twoslash'],
};
```

```ts
// source.config.ts
import { transformerTwoslash } from 'fumadocs-twoslash';
export default defineConfig({
  mdxOptions: {
    rehypeCodeOptions: {
      themes: { light: 'github-light', dark: 'github-dark' },
      transformers: [...rehypeCodeDefaultOptions.transformers, transformerTwoslash()],
      langs: ['js', 'jsx', 'ts', 'tsx'],   // pre-register, no lazy load in popups
    },
  },
});
```

```css
/* global.css */
@import 'fumadocs-twoslash/twoslash.css';
```

```tsx
// components/mdx.tsx
import * as Twoslash from 'fumadocs-twoslash/ui';
export function getMDXComponents(components?: MDXComponents) {
  return { ...defaultMdxComponents, ...Twoslash, ...components };
}
```

Usage: ` ```ts twoslash ` → inline type hovers. Optional `typesCache: createFileSystemTypesCache()` for perf.

### 7.5 Mermaid (⚠️ not built-in)
Fumadocs has NO Mermaid wrapper. The official recipe uses `mermaid` directly + `useTheme()` from `next-themes`:

```tsx
'use client';
import { useTheme } from 'next-themes';
const { resolvedTheme } = useTheme();
// ... uses mermaid.render with theme: resolvedTheme === 'dark' ? 'dark' : 'default'
```

**Conflict with our data-mode setup**: `useTheme()` returns `light`/`dark`, not `data-mode`. If we disable next-themes (DESIGN.md §1.6), we must rewrite this Mermaid renderer to read `data-mode` from `<html>`. Alternative: `beautiful-mermaid` (3rd-party, no theme coupling) — needs verification.

## 8. Deployment

| Target | Support | Notes |
|---|---|---|
| **Vercel** | ✅ Default | Next.js native. OG image generation works. |
| **Cloudflare Pages** | ⚠️ Via opennext.js.org/cloudflare | **Fumadocs doesn't work on Edge runtime**. Must use Node runtime. |
| **Cloudflare Workers** | ⚠️ Possible, tricky | See fumadocs discussion #2442. |
| **Static export** | ✅ `/docs/deploying/static` | Full static, CDN-hostable. **No server-side search** (use Algolia or pre-build Orama). **No dynamic OG images** (pre-compute or use static placeholders). |
| **Docker** | ✅ Official recipe | `WORKDIR` must include `source.config.ts` + `next.config.*` or MDX build fails. |

### 8.1 Edge runtime = NO-GO
> "Fumadocs doesn't work on Edge runtime." — explicit limitation.
→ On Cloudflare/Workers, force `runtime: 'nodejs'` on routes that use Fumadocs.

### 8.2 Static export caveats
- `output: 'export'` in next.config.mjs
- Search → external (Algolia) or pre-build Orama index to static files
- OG images → pre-compute at build time

### 8.3 Docker WORKDIR gotcha
```dockerfile
WORKDIR /app
COPY package.json pnpm-lock.yaml* source.config.ts next.config.* ./
```
Missing `source.config.ts` from COPY = MDX build fails inside container.

## 9. Gotchas — DeesseJS-specific

1. **`next-themes` conflict with DESIGN.md** ([[DESIGN.md §1.6]])
   - **Fix**: `<RootProvider theme={{ enabled: false }}>` then manage `data-mode` ourselves via small client component + FOUC script in `<head>` (the [[project-shadcn-final-state]] pattern).
   - Do NOT install `next-themes` ourselves — Fumadocs bundles it; just disable at RootProvider level.

2. **Theme preset choice**
   - Default scaffold uses `neutral.css` (grey). For DeesseJS design system: switch to `shadcn.css` + add `@import '@workspace/ui/globals.css'` BEFORE the fumadocs imports (per [discussion #1338](https://github.com/fuma-nama/fumadocs/discussions/1338)).
   - At apps/docs scope, packages/ui lives at repo root (`packages/`) and apps/docs is sibling of packages in pnpm workspace. Path: `../../packages/ui/globals.css` or alias `@deessejs-ui/globals.css` after wiring.

3. **Tailwind v4 `@source` path trap** (already known from [[project-shadcn-final-state]])
   - When wiring packages/ui components into Fumadocs MDX output, must add `@source` for `node_modules/fumadocs-ui/dist/**/*.js` and packages/ui components path.

4. **`prose` plugin conflict**
   - Fumadocs has its own Tailwind Typography fork. If we need `@tailwindcss/typography`, set `className: wysiwyg` to avoid conflict.

5. **Lucide icon names**
   - `lucide-react@1.20.0` in apps/docs — different from older 0.x line. Check actual installed version if upgrading. The scaffold already wires `lucideIconsPlugin()`.

6. **`.source/` is gitignored**
   - It's regenerated. Don't try to commit it. Already in `.gitignore` from scaffold.

7. **MDX runtime split**
   - `.source/server.ts` (RSC, server-only — `await` is used)
   - `.source/browser.ts` (client-side)
   - When importing from `.source/`, use the alias `collections/*` (set in tsconfig.json paths).

8. **Proxy.ts is Next.js 16 specific**
   - Replaces `middleware.ts`. Currently used for content negotiation (`/docs/test.md` → content.md). Don't refactor it to middleware.ts.

9. **`baseOptions()` returns a fresh object each call**
   - Currently `baseOptions()` is called per render. If we add expensive computation (e.g. async fetch), memoize it.

10. **Mermaid wants next-themes**
    - Built-in recipe uses `useTheme()`. If we disable next-themes for DESIGN.md, must rewrite Mermaid renderer to read `data-mode` attribute.

11. **Edge runtime not supported**
    - Restricts Cloudflare/Workers deploy options. If we go that route, must force Node runtime.

12. **Twoslash requires `serverExternalPackages`**
    - Forgetting this in `next.config.mjs` → bundler tries to bundle `typescript` → build fails or runtime errors.

13. **Static export breaks server features**
    - Search, OG images, dynamic content all break in static mode. Choose carefully.

14. **`sidebar.prefetch: true` is expensive on Vercel**
    - Fumadocs docs explicitly warn: "prefetch requests may cause a higher usage of serverless functions and Data Cache. It can also hit the limits of some other hosting platforms."
    - For DeesseJS (targeting Vercel per [[project-apps-docs-stack-decision]]), set `sidebar.prefetch: false` in `baseOptions()` from day 1. Reduces serverless function invocations significantly.

15. **Layout Tabs explicit array > `meta.json` root folders**
    - Per Way 2 in [[reference-fumadocs-api#4.4 Root folders = Layout Tabs]], passing `tabs={[{title, url}]}` as an explicit array to `<DocsLayout>` is cleaner than relying on `meta.json { root: true }` in content folders.
    - Why: content structure stays independent from navigation decisions. Reorganizing content (renaming folders, moving pages) doesn't break the top-level tab navigation.
    - Recommended for DeesseJS multi-product case (Template / Cloud / API) when we get there.

14. **OG images need Node runtime**
    - Default `@vercel/og` and similar packages need Node. On Cloudflare Edge, must adapt.

## 10. Customization patterns

### 10.1 Override MDX components
```ts
// src/components/mdx.tsx
import defaultMdxComponents from 'fumadocs-ui/mdx';
export function getMDXComponents(components?: MDXComponents) {
  return { ...defaultMdxComponents, ...components };
}
```

### 10.2 Customize frontmatter schema
```ts
// source.config.ts
import { z } from 'zod';
export const docs = defineDocs({
  dir: 'content/docs',
  docs: {
    schema: z.object({
      title: z.string(),
      description: z.string().optional(),
      icon: z.string().optional(),
      // custom fields
    }),
  },
});
```

### 10.3 Custom page tree transformation
Loader plugins can mutate the page tree (e.g. add separators, transform names). Examples: `lucideIconsPlugin`, `openapiPlugin`.

### 10.4 Add a new section as root folder (Layout Tabs)
```json
// content/<new-section>/meta.json
{ "title": "New Section", "root": true }
```
Fumadocs UI auto-renders the top-level tab.

### 10.5 Disable search entirely
```tsx
<RootProvider search={{ enabled: false }}>
```

### 10.6 Replace search with custom (e.g. Algolia)
```tsx
import { SearchDialog } from '@/components/algolia-search';
<RootProvider search={{ SearchDialog }}>
```
